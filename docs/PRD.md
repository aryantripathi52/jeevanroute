# Product Requirements Document (PRD)
## VitalRoute — AI-Powered Ambulance Routing SaaS

**Version:** 1.0  
**Date:** June 2026  
**Status:** Hackathon MVP  

---

## 1. Overview

### 1.1 Problem Statement
In India, ambulances routinely navigate to the nearest hospital — not the *right* hospital. Patients arrive at facilities that lack the required specialist, blood type, ICU beds, or equipment. This misrouting costs lives during the critical **Golden Hour** — the first 60 minutes after a medical emergency when intervention is most effective.

### 1.2 Solution
VitalRoute is a real-time, AI-powered hospital routing system for ambulance operators. It takes patient condition data, triages it using AI, and recommends the best-matched hospital based on specialty, proximity, and live bed availability — not just distance. It features a two-dashboard architecture: one for ambulance operators and one for hospital emergency staff, synced in real time.

### 1.3 Target Users
| User | Role |
|---|---|
| Ambulance Operator | Inputs patient data, receives hospital recommendations, navigates |
| Hospital Emergency Staff | Receives incoming patient alerts, accepts/declines, prepares |

## 3. Features

### 3.1 Ambulance Dashboard

#### Patient Intake Form
- Condition selector: Cardiac / Trauma / Neurological / Burns / Maternity / Respiratory / Other
- Severity scale: 1–10
- Patient details: Age, Blood Group, Known Allergies (optional)
- Auto-detected GPS location via browser geolocation

#### AI Triage Engine
- Sends patient data to Claude API
- Returns: required specialty, equipment needed, urgency level, recommended blood availability
- Response time target: < 5 seconds

#### Hospital Recommendation Screen
- Top 3 hospitals ranked by: specialty match score + ETA + distance
- Each card shows: Hospital name, Match %, Distance, ETA, Specialty badges, Bed status
- "Send Alert" button — notifies chosen hospital via Supabase Realtime
- Live map with route overlay using Google Maps Directions API

#### Auto-Failover on Decline
- If Hospital #1 declines, system detects via Supabase listener
- Re-fetches current ambulance GPS location
- Re-runs Claude triage with updated location + escalated urgency
- Auto-selects and alerts Hospital #2 without operator intervention
- UI shows: "🔴 Declined → 🔄 Rerouting → 🟢 Redirected to [Hospital]"
- New route loads automatically on map

---

### 3.2 Hospital Dashboard

#### Incoming Alert Card
- Patient condition summary (AI-generated, plain English)
- ETA countdown
- Patient: age, blood group, severity, allergies
- Ambulance current location on mini-map

#### Accept / Decline
- Accept: triggers confirmation to ambulance UI, starts prep checklist
- Decline: requires reason (ICU Full / No Specialist / Equipment Unavailable), triggers ambulance auto-failover

#### AI Prep Checklist
- Auto-generated by Claude based on patient condition
- Example for cardiac: "Prepare defibrillator · O- blood on standby · Cardiologist alerted · ICU Bay 3 cleared"
- Checklist is interactive — staff can check off items

#### Case Log
- All incoming alerts for the current shift
- Status: Pending / Accepted / Declined / Completed

---

### 3.3 Shared / System Features

| Feature | Description |
|---|---|
| Real-time sync | Supabase Realtime — both dashboards update instantly |
| Role-based entry | Landing page with "Ambulance" / "Hospital" selector |
| Live ETA | Updates every 30 seconds as ambulance moves |

## 4. User Flows

### 4.1 Normal Flow
```
Ambulance opens app
→ Fills patient intake form
→ AI triages condition
→ Top 3 hospitals displayed
→ Operator taps "Send Alert" to Hospital #1
→ Hospital dashboard receives alert
→ Staff reviews → clicks Accept
→ Ambulance sees "Confirmed ✅" + navigation starts
→ Hospital AI prep checklist activates
→ Patient arrives at correct facility
```

### 4.2 Decline & Auto-Failover Flow
```
Hospital #1 clicks Decline (reason: ICU Full)
→ Ambulance UI shows "🔴 Hospital Declined"
→ System fetches current ambulance GPS
→ Claude re-triages with updated location + escalated urgency
→ Hospital #2 auto-selected
→ Alert auto-sent to Hospital #2
→ Ambulance UI shows "🔄 Redirecting to [Hospital #2]..."
→ New route loads on map
→ Hospital #2 dashboard receives alert
```

## 5. Technical Architecture

### 5.1 Stack
| Layer | Technology |
|---|---|
| Frontend | Next.js 14 + Tailwind CSS |
| AI Triage | Claude API (claude-sonnet-4-6) |
| Database + Realtime | Supabase |
| Maps & Routing | Google Maps Platform (Maps JS + Directions + Places API) |
| Location | Browser Geolocation API |
| Deployment | Vercel |

### 5.2 Supabase Schema (MVP)

**cases table**
```
id, created_at, patient_condition, severity, age, blood_group,
allergies, pickup_lat, pickup_lng, current_lat, current_lng,
ai_triage_result, status, assigned_hospital_id, declined_hospitals[]
```

**hospitals table**
```
id, name, lat, lng, specialties[], available_beds, icu_available,
blood_types_available[], address, phone
```

**alerts table**
```
id, case_id, hospital_id, status (pending/accepted/declined),
decline_reason, created_at, responded_at
```

### 5.3 API Routes
```
POST /api/triage          → Claude API call, returns specialty + urgency
POST /api/recommend       → Google Places + scoring logic, returns top 3
POST /api/alert           → Insert alert into Supabase
POST /api/reroute         → Re-triage + re-recommend on decline
GET  /api/case/:id        → Fetch live case data
```
